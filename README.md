## How I build

- Start with the user workflow, not the model demo
- Design structured outputs and validation paths early
- Keep product surfaces clean enough that non-technical users can trust them
- Document architecture, ownership, tradeoffs, and next steps so the work is easy to evaluate
- Treat distribution, demos, and business constraints as part of engineering, not an afterthought
